## <a name="authentication-library"></a><span data-ttu-id="628d7-112">Libreria di autenticazione</span><span class="sxs-lookup"><span data-stu-id="628d7-112">Authentication library</span></span>

* <span data-ttu-id="628d7-126">I token hanno una durata breve, un'ora per impostazione predefinita, che limita la finestra di attacco.</span><span class="sxs-lookup"><span data-stu-id="628d7-126">Tokens have a short lifetime, one hour by default, which limits the attack window.</span></span> <span data-ttu-id="628d7-127">I token possono anche essere revocati in qualsiasi momento.</span><span class="sxs-lookup"><span data-stu-id="628d7-127">Tokens can also be revoked at any time.</span></span>
* <span data-ttu-id="628d7-128">Il token JWT autonomo offre garanzie al client e al server per il processo di autenticazione.</span><span class="sxs-lookup"><span data-stu-id="628d7-128">Self-contained JWTs offer guarantees to the client and server about the authentication process.</span></span> <span data-ttu-id="628d7-129">Un client, ad esempio, è in grado di rilevare e verificare che i token ricevuti siano legittimi e che siano stati emessi come parte di un determinato processo di autenticazione.</span><span class="sxs-lookup"><span data-stu-id="628d7-129">For example, a client has the means to detect and validate that the tokens it receives are legitimate and were emitted as part of a given authentication process.</span></span> <span data-ttu-id="628d7-130">Se una terza parte tenta di cambiare un token durante il processo di autenticazione, il client può rilevare il token cambiato ed evitare di usarlo.</span><span class="sxs-lookup"><span data-stu-id="628d7-130">If a third party attempts to switch a token in the middle of the authentication process, the client can detect the switched token and avoid using it.</span></span>
* <span data-ttu-id="628d7-131">I token con OAuth e OIDC non si basano sulla corretta presenza dell'agente utente per assicurarsi che l'applicazione sia protetta.</span><span class="sxs-lookup"><span data-stu-id="628d7-131">Tokens with OAuth and OIDC don't rely on the user agent behaving correctly to ensure that the app is secure.</span></span>
* <span data-ttu-id="628d7-132">I protocolli basati su token, ad esempio OAuth e OIDC, consentono di autenticare e autorizzare app ospitate e autonome con lo stesso set di caratteristiche di sicurezza.</span><span class="sxs-lookup"><span data-stu-id="628d7-132">Token-based protocols, such as OAuth and OIDC, allow for authenticating and authorizing hosted and standalone apps with the same set of security characteristics.</span></span>
